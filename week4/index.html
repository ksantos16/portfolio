OBJECT-ORIENTED PROGRAMMING

Notes:

Encapsulation: : the inner workings are kept hidden inside the object and only the essential functionalities are exposed to the end user

Polymorphism: various objects can share the same method, but also have the ability to override shared methods with a more specific implementation.

Inheritance: can take an object that already exists and inherit all its properties and methods. We can then improve on its functionality by adding new properties and methods.

Classes: to define a blueprint for an object.

CONSTRUCTOR FUNCTIONS

Notes:

The parentheses are not required when instantiating a new object using a constructor function. The following code would also achieve the same result:
const redDice = new Dice;
The parentheses are required, however, if any default arguments need to be provided.
For example, if we want to create another Dice object with four sides, we would have to add 4 as an argument, like so:
const whiteDice = new Dice(4);

CONSTRUCTOR PROPERTY

Notes:

We can use the constructor property to instantiate a copy of an object, without having to reference the actual constructor function or class declaration directly.

STATIC METHODS

Notes:

The static keyword can be used in class declarations to create a static method. These are sometimes called class methods in other programming languages.

PROTOTYPAL INHERITANCE

Notes:

Every class has a prototype property that is shared by every instance of the class. So any properties or methods of a class’s prototype can be accessed by every object instantiated by that class

The prototype object is live, so if a new property or method is added to the prototype, any instances of its class will inherit the new properties and methods automatically, even if that instance has already been created.
